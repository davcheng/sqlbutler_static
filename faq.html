SQL Butler

What is this?

SQLButler.com is an online SQL query assist tool that will generate CREATE and INSERT statements from any table you see.

Why would anyone use this?

While studying up on SQL for a technical interview, I wanted to get in some practice joining tables and selecting rows using nested queries. However, a few hours into my study session, I realized I hadn't learned anything because I had spent 90% my time creating and populating dummy tables rather than actually practicing JOINs and SELECTs (which is what I actually wanted to practice).

SQL Butler seeks to remedy this problem so that you can spend more time learning/living and less time data entry-ing/not living.

How do I use this?

  1. Boot up your favorite instance of SQL (e.g., mysql)
  2. Create a practice DB (CREATE DATABASE PRACTICE_DB;)
  3. Select the database (USE PRACTICE_DB;)
  4. Copy and paste any table you find in a SQL interview prep blog, w3schools article, or Excel spreadsheet into the Table Contents text area
  5. Name your table (if your table name is not one word, a mandatory SQL requirement, underscores will be used in place of spaces)
  6. Hit "Generate" and use the statements to create the table in your DB!

Why is every column a varchar(150)?

I figured if you're using this to practice joining tables, the data types don't really matter. Plus, the CREATE statement is pretty easy to edit/isn't horribly tedious to come up with (it's the insert statements that wear you down). And from a UI perspective, I didn't want to bog down the input area with an array of drop downs.

What if I love manually typing up CREATE/INSERT statements for large tables?

There is nothing for you here.
